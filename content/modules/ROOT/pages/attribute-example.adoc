== Copyable Command Blocks

Use `subs="attributes"` on source blocks to inject values into commands your users can copy-paste.

=== SSH to the Bastion

[source,bash,subs="attributes"]
----
ssh {bastion_ssh_user_name}@{bastion_public_hostname}
----

Password: `{bastion_ssh_password}`

=== Open the OpenShift Console

Visit the console at:

{openshift_console_url}

=== curl an App on the Cluster

[source,bash,subs="attributes"]
----
curl -sk https://my-app.{openshift_cluster_ingress_domain}/api/health
----

== Inline Usage

You can reference attributes inline in regular prose.
Your environment GUID is *{guid}* and the bastion is reachable at *{bastion_public_hostname}*.

== How It Works

In your AsciiDoc source, you write attribute references with curly braces:

[source,asciidoc]
----
ssh \{bastion_ssh_user_name}@\{bastion_public_hostname}
----

At build time, Antora replaces them with the values defined in `site.yml` (local dev) or injected from `user_data` (deployed environment).

TIP: Always provide sensible defaults in `site.yml` or `content/antora.yml` so your content renders correctly during local development. See xref:user-data.adoc[] for the full injection flow.
