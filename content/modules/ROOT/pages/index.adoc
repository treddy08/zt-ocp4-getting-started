= OpenShift Getting Started - OpenShift Concepts

Welcome to *OpenShift Getting Started -- OpenShift Concepts*!

In this lab, you'll deploy your first application to OpenShift and understand how all the pieces fit together.
The goal is simple: get a web app running and accessible from a browser -- using nothing but the command line.

== What you'll learn

In this lab, you will:

* Deploy a containerized application to OpenShift with a single command
* Understand the three core building blocks: Pods, Services, and Routes
* Use the `oc` CLI to inspect, create, and manage OpenShift resources
* Build an application from a container image to a publicly accessible URL

== Who this is for

This lab is designed for developers, platform engineers, and DevOps engineers who are new to OpenShift.
No prior Kubernetes or container experience is required.

*Experience level*: Beginner

== Prerequisites

Before starting this lab, you should have:

* Basic comfort with the command line
* A web browser
* No prior OpenShift or Kubernetes experience needed
